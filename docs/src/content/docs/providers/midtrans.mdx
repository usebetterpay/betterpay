Midtrans is a leading Indonesian payment gateway that provides Snap API for seamless checkout experience. This guide will walk you through the complete integration process.

## Overview

**Midtrans** provides:
- Snap API for hosted checkout page
- Direct API for custom checkout
- Virtual Account, E-Wallet, QRIS, Credit Card
- Comprehensive fraud detection

**Key Features:**
- SHA512 signature verification
- Webhook support with signature validation
- Sandbox environment for testing
- 3D Secure for credit cards
- Fraud detection system

## Prerequisites

Before you start, you need:

1. **Midtrans Merchant Account**
   - Register at [midtrans.com](https://midtrans.com)
   - Complete merchant verification

2. **API Credentials**
   - Server Key
   - Client Key

   Get these from your Midtrans dashboard: **Settings → Access Keys**

## Basic Setup

### 1. Configure Environment Variables

Create a `.env` file:

```bash
MIDTRANS_SERVER_KEY=your_server_key_here
MIDTRANS_CLIENT_KEY=your_client_key_here
```

### 2. Initialize Midtrans Provider

```typescript
import { betterPay } from '@betterpay/core';
import { MidtransProvider } from '@betterpay/midtrans';

const pay = betterPay({
  providers: [
    new MidtransProvider({
      serverKey: process.env.MIDTRANS_SERVER_KEY!,
      clientKey: process.env.MIDTRANS_CLIENT_KEY!,
      isSandbox: process.env.NODE_ENV !== 'production',
    }),
  ],
});
```

### 3. Create Payment

```typescript
// Create a payment link
const payment = await pay.createPayment({
  provider: 'midtrans',
  orderId: 'ORDER-123',
  amount: 150000,
  customerName: 'John Doe',
  customerEmail: 'john@example.com',
  customerPhone: '081234567890',
  description: 'Payment for Order #123',
  callbackUrl: 'https://yourapp.com/api/webhooks/midtrans',
  returnUrl: 'https://yourapp.com/payment/success',
  items: [
    {
      name: 'Product A',
      price: 100000,
      quantity: 1,
    },
    {
      name: 'Product B',
      price: 50000,
      quantity: 1,
    },
  ],
});

console.log('Payment URL:', payment.paymentUrl);
```

## Webhook Integration

### 1. Create Webhook Endpoint

<CodeGroup>
```typescript Next.js App Router
// app/api/webhooks/midtrans/route.ts
import { pay } from '@/lib/payments';
import { NextRequest, NextResponse } from 'next/server';

export async function POST(request: NextRequest) {
  try {
    const body = await request.text();
    const headers = Object.fromEntries(request.headers);

    // Verify webhook signature
    const isValid = await pay.verifyWebhook('midtrans', {
      body,
      headers,
    });

    if (!isValid) {
      return NextResponse.json(
        { error: 'Invalid signature' },
        { status: 401 }
      );
    }

    // Parse webhook
    const events = await pay.normalizeWebhook('midtrans', {
      body,
      headers,
    });

    // Process each event
    for (const event of events) {
      switch (event.name) {
        case 'payment.completed':
          await updateOrderStatus(event.payload.order_id, 'paid');
          break;
        case 'payment.expired':
          await updateOrderStatus(event.payload.order_id, 'expired');
          break;
        case 'payment.failed':
          await updateOrderStatus(event.payload.order_id, 'failed');
          break;
      }
    }

    return NextResponse.json({ success: true });
  } catch (error) {
    console.error('Webhook error:', error);
    return NextResponse.json(
      { error: 'Internal server error' },
      { status: 500 }
    );
  }
}
```


### 2. Register Webhook URL

In your Midtrans dashboard:
1. Go to **Settings → Configuration**
2. Set Payment Notification URL: `https://yourapp.com/api/webhooks/midtrans`
3. Save changes

### 3. Webhook Events

Midtrans sends these transaction status:

| Status | Description |
|--------|-------------|
| `capture` | Payment captured (credit card) |
| `settlement` | Payment settled |
| `pending` | Waiting for payment |
| `deny` | Payment denied |
| `expire` | Payment expired |
| `cancel` | Payment cancelled |

### 4. Webhook Payload

Example webhook payload:

```json
{
  "va_numbers": [
    {
      "bank": "bca",
      "va_number": "1234567890"
    }
  ],
  "transaction_time": "2023-12-16 15:00:00",
  "transaction_status": "settlement",
  "transaction_id": "abc123",
  "status_message": "Success, transaction is found",
  "status_code": "200",
  "signature_key": "abc123...",
  "payment_type": "bank_transfer",
  "order_id": "ORDER-123",
  "merchant_id": "G123456789",
  "gross_amount": "150000.00",
  "fraud_status": "accept",
  "currency": "IDR"
}
```

### 5. Signature Verification

The signature is generated using:

```typescript
signature = SHA512(order_id + status_code + gross_amount + server_key)
```

BetterPay automatically verifies this signature in `verifyWebhook()`.

## Checking Payment Status

You can check payment status programmatically:

```typescript
const status = await pay.checkStatus('midtrans', 'ORDER-123');
```

## Error Handling

### Common Errors

**Invalid Server Key**
```typescript
try {
  const payment = await pay.createPayment({ /* ... */ });
} catch (error) {
  if (error.message.includes('401')) {
    console.error('Check your server key');
  }
}
```

**Invalid Signature**
```typescript
const isValid = await pay.verifyWebhook('midtrans', { body, headers });
if (!isValid) {
  console.error('Webhook signature verification failed');
  return new Response('Invalid signature', { status: 401 });
}
```

## Best Practices

### 1. Always Verify Webhooks

```typescript
const isValid = await pay.verifyWebhook('midtrans', { body, headers });
if (!isValid) {
  return new Response('Invalid signature', { status: 401 });
}
```

### 2. Handle Idempotency

Process webhooks idempotently:

```typescript
// Check if webhook already processed
const existing = await db.webhooks.findUnique({
  where: { transactionId: event.payload.transaction_id },
});

if (existing) {
  return NextResponse.json({ success: true });
}
```

### 3. Use Snap Token for Custom UI

If you want to use your own UI:

```typescript
const payment = await pay.createPayment({
  provider: 'midtrans',
  orderId: 'ORDER-123',
  amount: 150000,
  // ...
});

// Use snap token for custom checkout
console.log('Snap Token:', payment.raw.token);
```

### 4. Monitor Fraud Status

Check fraud status in webhook:

```typescript
if (event.payload.fraud_status === 'deny') {
  console.error('Transaction denied by fraud detection');
  return;
}
```
